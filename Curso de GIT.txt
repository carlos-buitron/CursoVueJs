Tener instalado GIT
La secuencia es asi el "Working directory" contiene nuestro proyecto, cuando iniciamos GIT podemos enviar todo los archivos o ciertos archivos a un area temporal "staging area" que va guardar los cambios que realicemos en el "Working directory" y cuando queremos ya guardar esos cambios se hara en el "local repository" osea de forma oculta git va tener un repositorio con nuestras versiones de cambio.

Cuando queremos relacionar mi "working directory" con el "local repository" en nuestro computador vamos a iniciar con "git init", obs "sólo la primera vez al principio". Tener en cuenta que "remote repository" hace referencia por ejemplo "github", "git lab",etc.

Asegurar que los archivos del "working directory" tengan seguimiento para ello debo mostrar el estado de los archivos de mi "working directory" que no han sido registrados, entonces usar "git status", pero si tenemos ya archivos que fueron modificados, eliminados o creados entonces utilizar "git status -s".

Entonces para hacer un seguimiento de los archivos de mi "working directory" tenemos que mandar al area local "staging area" entonces usamos el comando "git add *" donde "*" puede ser "." ó el "nombreArchivo.extension", podemos validar haciendo "git status -" para validar si ya se encuentra en el "staging area" o área temporal porque perfectamente podemos ir monitoreando los cambios solo de un archivo en especifico o de todos los archivos perteneciente a mi "staging area". Para mandar algun archivo o todos los archivos de mi "staging area" o área temporal a nuestro repositorio "local repository" los "commit -m "escribir mensjae" son los que nos sirve para hacer una copia deel archivo en el "staging area"
